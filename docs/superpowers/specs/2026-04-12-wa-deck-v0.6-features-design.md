# WA-Deck v0.6 Features Design

**Date:** 2026-04-12
**Version:** 0.5.1 -> 0.6.0

## Overview

Four changes to WA-Deck: port voice messages from beta, fix schedule datetime defaults, add schedule popover to toolbar, and add template categories with grouped overlay.

---

## Task 1: Port Voice Messages from Beta

**Scope:** Direct code transfer from `Playground-Beta` (v0.6.0-beta.1) to `Playground` (v0.5.1). No design changes.

**Files to transfer:**

| File | Change |
|------|--------|
| `src/main.js` | Add `guessMime()` function + `pick-audio-file` IPC handler |
| `src/preload.js` | Add `pickAudioFile()` to context bridge |
| `src/renderer/index.html` | Add voice message button in toolbar + `<script>` tag for `voice-message.js` |
| `src/renderer/renderer.js` | Add `sendAudioAsVoiceMessage()` function + button event binding |
| `src/renderer/styles.css` | Add voice button styles + `.is-recording` pulse animation |
| `src/renderer/webview-scripts/voice-message.js` | New file (182 lines) — webview audio injection script |

**Constraints:**
- Button enabled only for WhatsApp accounts (not Telegram)
- Button enabled only when account is active, unfrozen, and chat is open
- Max audio file size: 16 MB
- Supported formats: `.ogg`, `.opus`, `.mp3`, `.wav`, `.m4a`, `.aac`, `.weba`

## Task 2: Schedule DateTime Default = Now

**Problem:** The datetime field in the schedule form is set once at app init via `nextSendAtLocal(5)` (now + 5 minutes). Since the app runs for days/weeks without restart, the default becomes stale.

**Solution:** Reset the datetime field to current time (no offset) every time the schedule form is opened or expanded:

- When the schedule `<details>` section in settings is expanded
- When the schedule popover (Task 3) is opened
- When the "Create message" section inside the popover is expanded

**Implementation:**
- Change call from `nextSendAtLocal(5)` to `nextSendAtLocal(0)` (or equivalent)
- Attach reset logic to `toggle` events on the relevant `<details>` elements and popover open events

## Task 3: Schedule Popover

**Trigger:** Click on `#tb-schedule` button in toolbar.

**Behavior:**
- Popover appears below the toolbar button (same pattern as weather/CRM popovers)
- Closes on click outside or repeat click on button
- Does NOT open or close the settings panel
- Settings panel retains its own schedule sections as-is (dual access)

**Structure:**
```
#schedule-popover
+-- <details> "Create message" (collapsed by default)
|   +-- Account/chat picker
|   +-- Text textarea
|   +-- Datetime input (default = now, reset on expand)
|   +-- Attachment button + attachment list
|   +-- "Schedule" button
|
+-- <details> "Scheduled (N)" (expanded by default)
    +-- List of scheduled messages
        +-- Status badge (pending/processing/failed/sent)
        +-- Truncated message text
        +-- Scheduled datetime
        +-- Edit / Cancel buttons
```

**Logic:**
- On popover open: fetch current list via `list-scheduled` IPC, update count `(N)`
- Create form uses `schedule-message` IPC (same as settings form)
- On expand of "Create message" `<details>`: reset datetime to now
- After create/cancel/edit: auto-refresh the list
- List updates reflect in both popover and settings panel (if open)

**Styling:**
- Width: 350-400px
- Max height with scroll for the list section
- Follows existing popover design: box-shadow, border-radius, theme colors
- Consistent with weather/CRM popover patterns

## Task 4: Template Categories

### Data Model

- Each template gains a `category` field (string, default `""`)
- Backward compatible: templates without `category` field are treated as `category: ""`
- No data migration needed

### Settings Form Changes

- New "Category" field added between title and text fields
- `<input type="text">` with `<datalist>` populated from existing categories
- Allows selecting existing category or typing a new one
- Category saved alongside template via existing `save-template` IPC

### Toolbar Overlay (`tq-overlay`) Changes

**Structure:**
```
#tq-overlay
+-- Category "Greetings" (collapsible <details>)
|   +-- Template "Welcome"
|   +-- Template "Hello"
|
+-- Category "Replies" (collapsible <details>)
|   +-- Template "Thank you"
|   +-- Template "Please wait"
|
+-- Category "Uncategorized" (always last)
    +-- Template "Old template"
```

**Behavior:**
- Categories rendered as `<details>` elements with arrow icon indicator
- Categories sorted alphabetically; "Uncategorized" always last
- Templates within each category sorted alphabetically
- Click on template inserts into active chat (existing behavior)
- Expand/collapse state of categories persists between overlay openings
- Templates retain their existing color labels
- If only one category exists or total templates are few: all expanded by default

**Styling:**
- Category header: accordion-style with arrow icon (triangular marker)
- Templates: indented within category, preserve existing color-coded appearance
- Consistent with overall app theme
