# Changelog

## 0.9.3

**Фикс подстановки имени.**

- **`{имя}` больше не подставляет «В сети»**: в шапке чата имя и строка статуса — соседние однотипные элементы, и извлекатель имени мог взять статус (невидимые направляющие метки WhatsApp обходили фильтр). Теперь перебираются все элементы шапки, метки срезаются, статусы отбрасываются расширенным фильтром — на любом языке интерфейса.
- Тот же дефект исправлен в переводчике: имя чата — ключ хранения языка контакта, и язык мог сохраняться под ключом «В сети» (выглядело как «слетает выбранный язык»).

## 0.9.2

### Перевод

- **Автоопределение языка контакта.** Дека запоминает язык контакта сама — по двум переведённым входящим (если язык не был выбран вручную). Кнопка «Перевести» сразу целится в нужный язык без дропдауна на каждый новый чат.
- **Батч-перевод входящих** (тумблер в «Интерфейсе», включён по умолчанию): пачка видимых сообщений уходит одним запросом — чат «прогружается» переводами быстрее, меньше шансов упереться в лимит. При сбое разбивки — прозрачный фолбэк по одному.
- **Cmd/Ctrl+Enter убран** (по фидбэку: неудобен и сбоил); панель переводчика возвращена к прежнему поведению. Перевод ускорен: keep-alive соединение, ретрай, кэш.

### Шаблоны

- **Умный порядок «/»-списка**: первыми — шаблоны на языке текущего контакта (по коду в названии/категории: «1de», «NL_morning»), затем часто используемые.
- **`//` — повторить последний шаблон** с подстановкой под текущий чат.
- **Чистое `{имя}`**: код контакта отрезается («Jan NL55» → «Jan») во всех подстановках — слэш, планировщик, «Вставить в чат».

### Прочее

- **Хаб: лента «Непрочитанные» упорядочена по аккаунтам** — в том же порядке, что в левой панели; внутри аккаунта — по числу новых.
- **Справка работает на macOS**: F1 не доходил до приложения при фокусе в чате — проведён тем же каналом, что и остальные хоткеи (на клавиатурах Mac — fn+F1, если F-ряд не переключён в стандартный режим).
- **CRM: автосохранение при закрытии** — несохранённая запись больше не теряется при закрытии драуэра крестиком/Esc («Отмена» по-прежнему откатывает).
- **Страховка сканера чатов**: запасные селекторы на случай следующего редизайна WhatsApp (иначе лента, избранные и важные умерли бы молча все разом).
- **Уборка папки данных**: аварийные копии store и остатки обновлений старше 30 дней удаляются автоматически.

## 0.9.1

**Крупный релиз: всё новое с версии 0.8.1.**

### Перевод сообщений

- **Cmd/Ctrl+Enter — перевод композера.** В поле ввода WhatsApp переводит весь набранный текст (с русского на язык контакта) и заменяет его прямо в поле; отправляете вручную. Выделять текст не нужно — кнопка «Перевести» тоже переводит весь композер.
- **Тумблер «Авто вх. → Русский» запоминается** для каждого аккаунта — переживает перезапуск, перезагрузку вебвью и гибернацию.
- **Кэш переводов** — повторяющиеся фразы и уже переведённые входящие показываются мгновенно.

### Шаблоны в поле ввода

- **Слэш-шаблоны.** Наберите «/» первым символом в композере — над полем откроется список шаблонов с фильтрацией по мере набора: ↑/↓ — выбор, Enter или клик — вставка. Подстановка `{имя}` / `{приветствие}` / `{дата}` / `{время}`.

### Клавиатура

- **Cmd/Ctrl+U — следующий непрочитанный** чат по всем аккаунтам; повторные нажатия разгребают инбокс без мыши.
- **Сквозные хоткеи прямо из чата:** Cmd/Ctrl+1..9 — переключение аккаунтов, Cmd/Ctrl+K — поиск, Cmd/Ctrl+T — шаблоны, Cmd/Ctrl+R — обновить, зум. Работают, даже когда курсор в поле ввода WhatsApp; учтена русская раскладка.
- **Автофокус поля ввода** сразу после переключения аккаунта — можно печатать без клика мышью.

### Глобальный поиск (Cmd/Ctrl+K)

- Ищет чат **по всем аккаунтам сразу**, кэш чатов сохраняется между запусками (спящие аккаунты ищутся по кэшу).
- **Поиск по номеру телефона** (без учёта `+`, пробелов, дефисов) и **транслитом** — «аня» находит «Anya», «мюллер» — «Mueller». Точные совпадения всегда выше.

### Стартовый хаб

- **Единый сегмент-контрол «Аккаунты | Непрочитанные | Избранные | Важные»** со счётчиками новых сообщений на каждой вкладке. Ленты показывают только чаты с новыми сообщениями — по всем аккаунтам, по избранным или по важным контактам.
- **Кнопка «Обновить все»** доступна прямо из хаба (угловая кнопка обновления).

### CRM

- **Поля карточки подстраиваются под текст** — «О контакте» и «Моя информация» растут по содержимому, кнопки и путь к файлу прижаты к низу.

### Прочее

- **Справка по F1** — все функции дека и горячие клавиши на одном экране.
- **Защита данных** — ежедневные снапшоты хранилища аккаунтов и защита от случайной потери логинов.
- **Автопроверка обновлений** раз в 12 часов.
- **Плавные анимации** панелей — интерфейс не «дёргается» при появлении баров.

### Платформа

- **Electron 43.1.0 → 43.1.1.**

## 0.9.0

**Фидбэк 0.8.9 + аудит 6 агентов.**

### Поведение по фидбэку

- **Cmd/Ctrl+Enter теперь только переводит** текст композера (с русского на язык контакта) и заменяет его — БЕЗ автоотправки. Отправка всегда ручная (обычный Enter). Заодно ушёл риск отправки не в тот чат.
- **Слэш-шаблоны вставляются чисто**: команда (`/de` и т.п.) больше не остаётся и не плодит дубли. Очистка поля идёт нативным вводом (Cmd/Ctrl+A + Backspace) — только он надёжно чистит редактор WhatsApp — затем вставляется шаблон в пустое поле.
- **Вкладки «Избранные» и «Важные» показывают счётчик** новых сообщений (как «Непрочитанные»); бейджи обновляются на любой вкладке хаба.
- **Кнопка обновления в углу активна в хабе** — обновляет все аккаунты (раньше была недоступна без активного аккаунта).

### Аудит: надёжность

- Планировщик не запускает повторную отправку при долгой (>5 мин) загрузке большого вложения — порог «зависшего» задания поднят до 20 мин.
- `important.js` показывает тост, если чат избранного/важного не открылся (была асимметрия с избранными).
- Поля настроек (`autoTranslateAccounts`, `worldClocks`, `contactLangs`) санитизируются и при загрузке store, не только при сохранении.
- Пустой результат перевода больше не кэшируется навсегда.
- Слэш-Enter не конфликтует с Cmd+Enter переводчика.

### Аудит: Windows

- Удаление аккаунта и запись store ретраятся при залоченных антивирусом/индексатором файлах (EBUSY/EPERM) — сессии и данные не «застревают».
- Вложения с диска D:/сетевых шар больше не отклоняются, если выбраны через системный диалог.

### Аудит: macOS-апдейтер

- Проверка sha512 обязательна (пустой хэш в манифесте = отказ, а не пропуск).
- Приложение гарантированно перезапускается после обновления в любом исходе; ход установки пишется в лог.
- **Известно (нужно решение владельца):** сборки ad-hoc-подписаны, поэтому после авто-обновления возможны повторный QR-скан и запрос доступа к микрофону. Полностью лечится подписью Developer ID + нотаризацией (Apple Developer, $99/год).

### Производительность и уборка

- Переводчик в фоновых аккаунтах тикает реже (idle-backoff) — меньше постоянной нагрузки CPU при 10-20 аккаунтах.
- Удалён мёртвый код: idle-sweeper, модальный флоу обновления; убран декоративный «блик» на пунктах меню настроек.

### UX-полировка

- Панель «Сейчас важно» (избранные/важные) и бар переводчика больше не «дёргают» интерфейс: появляются и сворачиваются плавной анимацией высоты вместо резкого скачка на 44px.

## 0.8.9

**Фикс-пасс по фидбэку 0.8.8.**

- **Вкладки «Избранные» и «Важные» в хабе теперь показывают только непрочитанные** — как «Непрочитанные», но по избранным / важным контактам (чаты с новыми сообщениями), а не весь список аккаунтов. Пусто → «Нет непрочитанных у избранных/важных».
- **Слэш-шаблон вычищает команду**: после выбора шаблона набранная команда (например `/de`) больше не остаётся приклеенной перед текстом — поле полностью заменяется на шаблон (надёжная очистка `selectAll`+`delete` и проверка на точное совпадение).
- **Кнопка «Обновить все» убрана снизу хаба**: угловая кнопка обновления (справа от логотипа) теперь работает и в хаб-меню — обновляет все аккаунты, а не только активный.

## 0.8.8

**Фикс-пасс по фидбэку 0.8.7.**

- **Слэш-шаблоны снова работают целиком**: стрелки, Enter и клик. Lexical-обработчик WhatsApp на document-capture глушил наши клавиши — слушатель переехал на window-capture (срабатывает раньше); вставка выбранного шаблона получила верификацию с фолбэками (paste-событие → beforeinput), как у переводчика.
- **Cmd+Enter: убран ложный «Перевод не вставился»**. Композер применяет вставку асинхронно — первая проба была ложно-отрицательной: тост пугал, а Enter не отправлялся. Теперь повторная проба через паузу, после подтверждения — отправка; тост убран совсем (при реальном сбое исходник просто остаётся в композере).
- **CRM: оба поля подстраиваются под текст** — «О контакте» больше не зажат со скроллом, «Моя информация» не растягивается на всю высоту (field-sizing). Кнопки и путь к файлу прижаты к низу, длинные карточки скроллятся.
- **Хаб: один сегмент-контрол вместо двух рядов** — «Аккаунты | Непрочитанные | Избранные | Важные»; дублирующая строка фильтров у списка аккаунтов убрана.
- **Хаб: кнопка «Обновить все»** — рядом с «Добавить WhatsApp/Telegram» (то же действие, что в контекстном меню обновления).
- **Настройки: «Проверить обновление» и «Папка данных» вернули крупный размер** — в одну величину с карточками меню (компакт-пасс 0.8.5 ужал их слишком сильно).

## 0.8.7

**Честность и скорость: 7 багфиксов тихих отказов + сквозные хоткеи + ускорение переходов. Ноль нового UI.**

### Багфиксы (тихие отказы)

- **Cmd+Enter больше не молчит при фейле**: перевод не удался → тост «сообщение НЕ отправлено»; перевод не вставился → «отправьте вручную».
- **Планировщик не рапортует «sent» для ненапечатанного текста**: перед Enter проверяется, что текст реально в композере (промах клика по композеру раньше давал ложный успех); фейл ретраебелен.
- **Cmd+U и избранные не «съедают» чат при неудачном переходе**: если чат не открылся, строка возвращается в ленту, подавление снимается, показывается тост.
- **«Изменить» у серии больше не убивает повтор**: форма редактирования заполняет селекты повтора/джиттера из задания, а после сохранения сбрасывает их (раньше следующее разовое молча наследовало «каждый день»).
- **Esc снова работает внутри WhatsApp**: когда ни один слой дека не открыт, Esc пробрасывается в чат (снять цитату ответа, закрыть поиск — раньше дек его проглатывал).
- **Cmd+U не прыгает по устаревшему списку**: данные старше 15 секунд пересканируются перед прыжком.
- **Слэш-шаблоны не показывают старый текст после правки**: кэш списка сбрасывается при закрытии попапа (был залипаний до 30 секунд).

### Сквозные хоткеи и автофокус

- **Cmd+1..9, Cmd+K, Cmd+T, Cmd+R и зум теперь работают из композера чата** — раньше они умирали, как только курсор оказывался в поле ввода WhatsApp (перехват на уровне окна, как у Cmd+U; русская раскладка учтена).
- **Автофокус композера после переключения аккаунта**: после Cmd+N можно сразу печатать, клик мышью больше не нужен.

### Скорость

- **Переход в чат заметно быстрее**: фиксированные паузы в цепочке «поиск → клик → чат» заменены опросом готовности (поиск-путь ~2 c → ~1 c, клик по списку ~0.6 c → ~0.3 c); печать имени вдвое быстрее.
- **Cmd+K открывает чат быстрым путём**: сначала клик по видимой строке списка (~0.5 c), поиск WhatsApp — только фолбэк.
- **Cmd+K ищет транслитом и ранжирует**: «аня» находит «Anya», «мюллер» — «Mueller»; точное совпадение всегда выше подстрочных; для запроса-номера активный аккаунт первый в списке.
- **Кэш переводов**: повторные фразы и ре-переводы после перезагрузки webview мгновенны и не тратят запросы.
- **Фоновые сканы непрочитанных дешевле на ~85%**: аккаунты с нулевым счётчиком в заголовке вкладки не сканируются инъекцией (страховочный полный скан раз в 30 с — из-за muted-чатов); поллинг счётчиков 5 с → 10 с.
- **Анимации панелей ускорены до «мгновенных»** 120-160 мс (было 220-300).

### Прочее

- **Тумблер «Авто вх. → Русский» переживает рестарт**: состояние хранится per-account и восстанавливается после перезапуска, перезагрузки webview и гибернации.
- **Автоперевод входящих не умирает молча**: 5 фейлов перевода подряд — тост «проверьте сеть/лимит API» (не чаще раза в 5 минут).
- **Проверка обновлений раз в 12 часов** (была одна за жизнь процесса — при аптайме в недели о релизах узнавали случайно).
- **Лента непрочитанных ставит важных и избранных выше** болталок с большим счётчиком — тот же порядок, что у закреплённой полосы.

## 0.8.6

**Cmd/Ctrl+Enter — «перевести и отправить».**

- В композере чата Cmd/Ctrl+Enter переводит весь набранный текст (выделять не нужно) на язык контакта и отправляет одним действием. Enter уходит нативным вводом и только после подтверждённой вставки перевода — непереведённый исходник улететь не может; при сбое перевода текст остаётся в композере. Кнопка «Перевести» тоже больше не требует выделения: без него переводится весь композер (с заменой текста, а не дописыванием). Хоткей — в справке F1.

## 0.8.5

**Cmd/Ctrl+U «следующий непрочитанный», защита store от затирания, компактная панель настроек.**

- **Cmd/Ctrl+U — следующий непрочитанный.** Прыжок в чат с наибольшим числом непрочитанных по всем аккаунтам; повторные нажатия разгребают инбокс дальше без мыши. Работает и когда курсор в композере чата (перехват на уровне окна, как у Esc), и при закрытом хабе (скан по требованию). Учтена русская раскладка. Хоткей добавлен в справку F1.
- **Защита store от потери аккаунтов.** Первая запись каждого дня снимает снапшот `wa-deck-store.json.snap-YYYYMMDD` (хранится 7 последних). Wipe-guard: если сессия загрузилась с аккаунтами, а новая запись внезапно содержит 0 (класс отказов, который затирал и primary, и .backup разом), запись блокируется, подозрительный payload уходит в `.suspect`; явное удаление последнего аккаунта проходит как раньше.
- **Компактная панель настроек.** Меню, шапки секций, поля, кнопки и тумблеры ужаты до плотности «Мировых часов» (контролы ~26px): детальные секции потеряли лишние 16px-отступы и 12px-зазоры, кнопки в панели 26px вместо 30, CRM-драуэр приведён к той же плотности.

## 0.8.4

**Слэш-шаблоны, единая лента непрочитанных, справка F1, дизайн-пасс.**

- **Дизайн-пасс по аудиту.** Нативные виджеты (селекты, календарь, скроллбары) больше не светятся светлой OS-темой поверх тёмной; селекты получили единый вид с полями ввода; быстрые кнопки времени планировщика — ровно в один ряд; вкладки хаба — сегмент-контрол, визуально отличимый от фильтров; шапка хаба выровнена по одной оси; панели поиска и справки — на общих токенах; клавиатурный фокус для новых элементов.

- **Дизайн-пасс, часть 2: настройки и CRM.** «Мировые часы» переведены на общие токены: строки списка, поля и селекты — в одном стиле с остальными формами (свой шеврон вместо OS-стрелки), кнопка удаления — тихая, подсвечивается красным только при наведении, «+ Добавить» подсвечивается акцентом. Селект единиц в «Погоде» — в общем стиле. CRM-панель больше не растягивает секции по всей высоте с провалами: контент сверху, «Моя информация» занимает свободное место, кнопки и путь к файлу прижаты к низу. Шеврон селектов больше не пропадал в светлой теме и при наведении.

- **Слэш-шаблоны в поле ввода.** Наберите «/» первым символом в композере WhatsApp — над полем появится список шаблонов с фильтрацией по мере набора: ↑/↓ выбор, Enter/клик вставляет (с подстановкой {имя}/{приветствие}/{дата}/{время}), Esc закрывает.

### Root cause

WhatsApp Web 30 апреля задеплоил заметно более тяжёлый JS-бандл — navigation duration вырос с ~1с до ~3.4с, плюс появилась новая инструментация `responsiveness-db.unresponsiveness-events-v2` (теперь они сами трекают unresponsiveness события). На каждое `display:none → display:flex` переключение `<webview>` Chromium на Windows (ANGLE → DirectX 11) выбрасывал GPU-слой webview-а и при возврате должен был перерастеризовать его заново — это и давало «исчезновение иконки на 2-3 секунды». macOS Metal делает re-attach незаметно.

### Fix

Переключение теперь через `opacity: 0/1` + `pointer-events: none/auto` + `transform: translateZ(0)` (форсирует свой compositor layer) вместо `display: none`. GPU-слой держится горячим у всех webview одновременно — переключение измерено в **~13мс** вне зависимости от платформы.

Trade-off минимальный: каждый неактивный webview добавляет ~5MB GPU-памяти на компоновщик (для 10 аккаунтов — ~50MB). Логика keep-alive (Chromium-флаги, `setBackgroundThrottling(false)`, `visibilityState` spoof в `keep-alive.js`) не изменилась — всё ещё держит WebSocket WhatsApp Web живым в фоне.

## 0.7.8

**Стабильность и производительность.** Лаги при переключении между аккаунтами заметно сократились, добавлена опциональная гибернация неактивных webview, чистка HTTP-кэша и orphan partitions, корректная реакция на пробуждение системы. Бамп Electron до 41.3.0. Современнее установщик для Windows.

### Производительность

- **`renderScheduled` не дёргается на каждый switch.** Список отложенных глобальный (не per-account) — пересоздаётся только при реальном изменении данных (create/cancel/edit), а не при каждом переключении аккаунта. Раньше каждый switch делал IPC + DOM-rebuild списка из 30 элементов.
- **`updateHubClocks` переиспользует DOM-узлы** вместо `innerHTML = ''` и пересоздания каждые 30 секунд. Reflow на хабе устранён.
- **Periodic safe HTTP cache cleanup** — каждые 6 часов и через 5 минут после старта чистится только Chromium HTTP/code cache по партициям. IndexedDB, localStorage, cookies, serviceworkers НЕ трогаются — логин WhatsApp/Telegram сохраняется.
- **Cleanup orphan partitions при старте.** Директории `userData/Partitions/wa_*` и `tg_*` от уже удалённых аккаунтов удаляются в начале сессии. На длительных проектах — гигабайты свободного места и меньше disk IO.

### Долгоживущие сессии

- **`powerMonitor.suspend/resume`** — после простоя системы дольше 10 минут (закрытая крышка ноута, sleep) все webview мягко перезагружаются с интервалом 250 мс. Раньше WhatsApp оставался в zombie-состоянии и требовал ручного клика на каждом аккаунте.
- **`did-fail-load` фильтр.** ERR_ABORTED (-3) и ERR_NETWORK_CHANGED (-21) на main frame больше не показывают тост «Не удалось загрузить» — это нормальные транзиентные ошибки при VPN/Wi-Fi switch.

### Гибернация неактивных аккаунтов (opt-in)

- Новая настройка **Интерфейс → Гибернация неактивных аккаунтов**. Варианты: Выкл (по умолчанию), 30 мин, 1 ч, 2 ч, 4 ч.
- При включении: webview-ы простаивающих **не закреплённых** и **не активных** аккаунтов выгружаются из памяти (~80–150 MB на каждый), на карточке появляется 💤.
- Trade-off честно: пока аккаунт «спит», входящие не приходят real-time. Но не теряются — WhatsApp буферизирует их на сервере и выдаёт пачкой при следующем клике на аккаунт. Логин не пропадает (кукки в персистентной partition).
- Scheduled-отправка работает даже для гибернированных — планировщик автоматически разбудит webview за несколько секунд до отправки.

### Update toast

- Когда обновление обнаружено, теперь появляется **компактный toast в правом нижнем углу** (раньше — крупная блокирующая модалка). Не закрывает чат, отслеживает прогресс загрузки. Когда готово к установке — кнопка «Установить» прямо в toast'е.

### Переводчик

## 0.7.6

**Hotfix для CRM** — устранена критическая data-integrity ошибка, при которой CRM-запись одного контакта могла «переезжать» на другой контакт при его открытии.

### Critical fix

- **Фикс «CRM одного контакта у всех».** В `main.js:migrateLegacyCrmContactFile` — при загрузке CRM для контакта, если файл ещё не существовал, функция fuzzy-находила «самый богатый» CRM-файл в директории аккаунта и копировала его на путь нового контакта, **удаляя оригинал**. Изначально функция была задумана как one-shot миграция после смены формата имени файла, но срабатывала и на обычных новых контактах — и CRM-запись постоянно «блуждала» между контактами. Проявлялось так: открываешь контакт A с CRM, потом B — B видит CRM от A; открываешь C — C видит его же, и т.д. Каждый раз одна и та же запись «переезжает» на только что открытый контакт.
- **Исправление:** добавлен строгий name-match — кандидат на миграцию принимается только если его embedded `Контакт:`-поле точно совпадает с именем целевого контакта (после normalize). Посторонние файлы теперь не трогаются.

### Восстановление повреждённых данных

Если до установки 0.7.6 ты уже открывал CRM у нескольких контактов и видел эффект «один CRM на всех» — данные в файлах на диске могут быть перемешаны. Путь директории: `~/Library/Application Support/wa-deck/crm-contacts/<accountId>/`. Внутри каждого `.txt`-файла первая значимая строка — `Контакт: <имя>`, это фактический владелец данных. При необходимости можно вручную переименовать/перенести содержимое по корректным контактам.

## 0.7.0

**Первый polished-релиз** — серия аудитов дизайна, кросс-платформенного ревью и hardening-фиксов привела к существенной переделке UX, улучшению производительности на слабых машинах и укреплению security-модели.

### Переделка интерфейса
- Единый drawer-паттерн для всех настроек (шаблоны, отложенные сообщения, мировые часы, погода, интерфейс) с back-навигацией и иконкой в header. Убраны полноэкранные оверлеи шаблонов и floating schedule-popover — всё в правой панели.
- Редактирование шаблона теперь inline в том же drawer'е (раньше открывалось отдельное окно). В header'е формы: заголовок `Новый шаблон / Редактирование · <имя>` + кнопка закрытия.
- Список шаблонов рендерится lazy: категории свёрнуты по умолчанию, items появляются только при expand. Для 500 шаблонов — ~20 DOM-узлов на старте вместо 500+.
- Отложенные сообщения: chunked render (первые 30 + кнопка «Показать ещё (N)») — не виснет при 200+ записях.
- Chat picker в отложенной отправке показывает только чаты **активного** аккаунта (убрано переключение между аккаунтами внутри модала). Время отправки теперь всегда свежее при открытии drawer'а.
- Footer дашборда: `Добавить WhatsApp` (filled primary зелёный) / `Добавить Telegram` (outline) / шестерёнка (icon-only) — выстроена иерархия действий.
- Toolbar: компактные часы (убран лишний chip `MSK·KYIV·BER·LON`, зоны остались в hover-popover). Status-pill — теперь оформлен с border/background и auto-clear через 3с; debug-инфо (Electron/Chromium версии) убрана.
- Темы: dark и light объединены единым визуальным языком. Фон дашборда — subtle dot-grid вместо гор/солнца/звёзд/облаков. Новая палитра account-цветов (8 различимых оттенков без дубликатов красного).
- Семантические signal-токены: `--signal-error`, `--signal-warn`, `--signal-info`, `--signal-ok`. Красная точка «Нет активного WhatsApp» заменена на amber warn, unread-индикатор в toolbar — на accent вместо red.

### Доступность
- Глобальный `prefers-reduced-motion: reduce` — все анимации схлопываются при системной настройке «Уменьшить движение».
- Focus-visible fallback для всех interactive-элементов (2px green ring).
- Hit-area 32×32 у всех close-кнопок (× в панелях, модалах).
- Контраст toolbar-меток повышен (10px → 11-12px, темнее серый).

### Производительность
- **Lazy-load webviews:** при старте приложения webview'ы создаются с задержкой 400мс между каждым — UI отзывчив сразу, unread-счётчики заполняются в фоне. Для 30 аккаунтов стартовая загрузка быстрее и плавнее.
- **Idle-suspend webviews:** при бездействии >15 минут webview уничтожается, сохраняя session-cache (login остаётся). При повторной активации — мгновенная reload. Экономия памяти для power-users: ~2GB+ при 30-аккаунтном сетапе.
- Translator MutationObserver: batched flush через 120мс вместо fire-on-every-mutation — снижение CPU на слабых GPU.
- Cache-limits: CRM hover cache LRU-200, translator translation cache LRU-500.
- Single-entry chatPickerCache (Map → одна запись для активного аккаунта).

### Исправления багов
- Иконка удалённого аккаунта больше не висит в sidebar до рестарта — `renderAccounts()` + `updateHubDashboard()` вызываются всегда после удаления.
- Время в «Отложенная отправка» по умолчанию = текущее системное при каждом открытии раздела (раньше отставало на несколько минут).
- Toggle Переводчика/CRM hover больше не сбрасывает температуру погоды в toolbar.
- CRM hover popover теперь flip'ается влево если не помещается справа, z-index выше WhatsApp webview content, transition на позицию.
- Hero-card на дашборде не растягивается в пустое пространство при малом числе аккаунтов (`height: fit-content`).
- Sidebar `+` кнопка добавления аккаунта — sticky bottom, не исчезает при длинном списке.
- Sidebar и hub: счётчик непрочитанных не слипается с именем, цветная полоса карточки убрана (цвет только в chip-аватаре).

### Security & Data integrity
- **Path traversal защита:** иконки аккаунтов копируются в `userData/icons/<id>_<hash>.ext` с size-limit 10MB; вложения отложенных сообщений проверяются по whitelist (только внутри `$HOME`, блок на `.ssh/.gnupg/.aws/.docker/Keychains`), case-insensitive на Windows/macOS.
- **Validation IPC:** централизованные `LIMITS.*` для всех user-supplied строк (`ACCOUNT_NAME:60`, `TEMPLATE_TEXT:50000`, `MESSAGE_TEXT:65000`, `TRANSLATE_TEXT:20000` и т.п.); quota-limits (`TEMPLATES_PER_USER:5000`, `SCHEDULED_PER_USER:2000`).
- **Color validation:** `set-account-color` принимает только hex (`^#[0-9a-f]{3-8}$`).
- **Auto-update:** `autoInstallOnAppQuit=false` — обновление только по явной команде пользователя. Supply-chain risk (compromised GitHub) снижен.
- **Store safety:** `STORE_MAX_SIZE=50MB` против OOM при corrupt JSON; atomic save через `.backup` — recovery при power-loss.

### Кросс-платформа
- Windows NSIS: `oneClick: false`, `allowToChangeInstallationDirectory: true` — install без admin, с выбором папки и созданием shortcuts.
- **Windows portable** target добавлен — `.exe` без install, для restricted-окружений.
- Electron **41.2.0 → 41.2.2** (+23 CVE backports, fix PDF save, AudioWorklet nodeIntegration, always-on-top events). Chromium 146.0.7680.188.
- `npm audit fix` — обновлены транзитивные уязвимости (lodash, brace-expansion, picomatch).
- Удалён дубликат CI workflow (`build-windows.yml`) — race на артефактах устранён.

### Техдолг / cleanup
- Удалено ~900 строк dead-кода: `.hub-clouds/.hub-star/.hub-cloud-drift`, `.tq-*` (~17KB CSS), fullscreen template-edit модал, schedule-popover, mountain SVG фоны, `Playground-Beta/` каталог (~600MB).
- Устаревшие memory/cache Map'ы упрощены до single-entry.
